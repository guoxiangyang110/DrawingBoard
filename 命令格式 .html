易全解批改格式说明文档

INIT命令

每个批改片段的起始元素，time为起始时间，config带有页面宽高、录音地址。 批改录制开始录音随即开始，录音长度有文件自己确定，这里不强制录音文件为 本地文件，如果录音存为网络形式，需要预加载过程。

[
    {
        "name":"INIT",
        "time":[UTC time stamp]
        "config":{
            "page_width":0.4,
            "page_height":0.3,
            "voice_path":"voice/201705081020-234.mp3"
        }
    },
    ...
]

ADD命令

表示用户添加了一条轨迹线，id为全局统一编制，不可重复;penColor为32位色值，penWidth为基于页面大小的笔迹宽度 beginTime,endTime表示当前轨迹的起止时间，在INIT之后，时间可以用相对时间表示，这里一定程度缩短了录制文件长度

[
    ...
    {
        "name":"ADD",
        "time":[UTC time stamp],
        "trace":{"id":3,
                "penColor":-3285959,
                "beginTime":18506,
                "penWidth":0.0011534225195646286,
                "endTime":19084,"pathType":1,
                "tracePointList":
                    [   {"x":0.08452,"y":0.20009,"time":18506},
                        {"x":0.09114,"y":0.19870,"time":18619},
                        {"x":0.11985,"y":0.20144,"time":18652},
                        ...
                        {"x":0.17209,"y":0.26508,"time":19084}]
                }
    }
    ...
]

RMV命令

表示用于删除了某轨迹

id表示作用目标trace的id值

{"id":2,"time":23759,"name":"RMV"}

POP命令

表示用户“找回”了某轨迹。

id表示作用目标trace的id值

{"id":2,"time":24326,"name":"POP"}

CLEAR命令

表示所有轨迹被清空，包括回退堆栈

{"time":26277,"name":"CLEAR"}

小结

上述命令除了ADD都是瞬时命令，time属性都表示命令的发送时间也就是诞生时间。 ADD命令的时间，指向了命令的开始时间，也就是轨迹被创建的开始时间。 考虑到可能存在的格式化因素，两条命令之间以连续两个换行符号作为分割，这样既可以使用全部加载也可以使用渐进加载；为了兼容JSON的数组，命令序列化后第一行为 [ ，最后一行为 ],命令末尾使用 , 进行分割，渐近加载需要自己处理。 典型实例：

[
    {"name":"INIT",...},
    {"name":"ADD",...},
    {"name":"ADD",...},
    {"name":"ADD",...},
    {"name":"ADD",...},
    ...
    {"name":"RMV",...},
    {"name":"POP",...},
    ...
    {"name":"INIT",...}, //可以再次开始一个批改流程，但是**本句注释**不允许出现
    ...
]
